## Milestone Summary

**Key Decisions:**

- **Unaided-first protocol** — 122B attempted task2 (the `.fsl` lexer) with **no** lexer body and **no** FsLex API hints in the prompt. Hypothesis: bigger model can do what 35B could not. Outcome: confirmed; 122B wrote structurally valid FsLex on first attempt (rule/parse, no `%%`).
- **Honest timing** — the legacy "~14–32s/cycle" figure from v1 turned out to be a pre-run prediction (from 06-RESEARCH.md §1.3), not a measured value. v1.1 derived actual per-call averages from both runs' JSONL timestamps (35B ≈ 5.3s, 122B ≈ 6.3s) and explicitly corrected the stale figure both in 부록 C §3 (with disclaimer) and in the README.
- **Setup-asymmetry disclosed** — 35B had the `.fsl` lexer scaffolded for it; 122B did not. This is stated at the top of 부록 C (§실험 설계) before any capability claim, and reinforced inline in §1.
- **No manual edits** — same honesty discipline as v1; every ActionEvent across all 5 122B JSONLs has `source=agent`. The 9-iteration FsLex API fix sequence in task5 was entirely the agent's own.
- **Beginner UX callouts** — added as an out-of-band edit (no formal phase) after the user identified that 초보자가 흐름을 따라가기 어려웠다. Pattern: `📨 사용자 프롬프트 → ⚙️ 내부 프로세스 → ✅ 결과` blockquote groups, applied only where a real run cycle is described (not as a forced pattern on conceptual paragraphs).
