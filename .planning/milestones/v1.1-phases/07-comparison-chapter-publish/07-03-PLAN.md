**Step 3 — Wait for Actions deploy to finish.** Use `gh` CLI to poll the workflow run for the pushed SHA. Typical deploy time is 2–3 minutes; allow up to 8 minutes before declaring failure.

```bash
# Find the run for the pushed SHA
sleep 15  # let GitHub register the push
for i in $(seq 1 32); do
  STATUS=$(gh run list --workflow=deploy.yml --branch=main --limit=10 --json headSha,status,conclusion \
    | python3 -c "import json,sys; d=json.load(sys.stdin); m=[r for r in d if r['headSha']=='$HEAD_SHA']; print(m[0]['status']+':'+(m[0]['conclusion'] or '')) if m else print('not-found')")
  echo "[$i] $STATUS"
  case "$STATUS" in
    completed:success) echo "DEPLOY OK"; break ;;
    completed:failure|completed:cancelled|completed:timed_out)
      echo "DEPLOY FAILED: $STATUS"
      gh run view --log-failed --json url --jq .url 2>/dev/null || true
      exit 1 ;;
    not-found) ;;  # workflow not registered yet; keep polling
    *) ;;  # in_progress / queued
  esac
  sleep 15
done

if [ "$STATUS" != "completed:success" ]; then
  echo "FAIL: workflow did not complete success within ~8 min"; exit 1
fi
```

If `gh` is not authenticated, surface the gh-auth prompt verbatim and STOP — this is an authentication gate. Re-run after auth.
