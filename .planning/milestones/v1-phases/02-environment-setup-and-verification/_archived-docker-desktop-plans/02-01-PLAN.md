<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Install Docker Desktop and enable the default socket</name>
  <what-built>Nothing automated yet — Docker Desktop is a GUI macOS app with no headless install path in this environment, and enabling the socket requires the Settings UI.</what-built>
  <how-to-verify>
    Complete these manual steps, then Claude will verify:
    1. Download Docker Desktop for macOS Apple Silicon (ARM64) from https://www.docker.com/products/docker-desktop/
    2. Install it (drag to Applications) and launch Docker Desktop. Grant any permission prompts (privileged helper).
    3. In Docker Desktop -> Settings -> Advanced, enable "Allow the default Docker socket to be used", then click Apply & Restart.
    4. Wait until the Docker Desktop whale icon shows "Docker Desktop is running".

    After you confirm, Claude runs these to verify (expected output noted):
    - `docker info | grep "Server Version"`  -> expected: a line like `Server Version: 28.x.x`
    - `docker run --rm hello-world`           -> expected: contains `Hello from Docker!`
  </how-to-verify>
  <resume-signal>Type "docker ready" once Docker Desktop is running and the socket is enabled, or describe any error.</resume-signal>
  <verify>After resume: `docker info | grep "Server Version"` prints a Server Version line, AND `docker run --rm hello-world` output contains "Hello from Docker!". If either fails, re-check the socket setting and that Docker Desktop is fully started (not still booting).</verify>
  <done>`docker info` reports a Server Version and `docker run --rm hello-world` prints the Hello from Docker! message.</done>
</task>

<task type="auto">
  <name>Task 2: Install Python 3.12 and the OpenHands 1.7 CLI via uv</name>
  <files>(no repo files; installs to uv-managed ~/.local/bin)</files>
  <action>
    Run, in order:
    1. `uv python install 3.12`  — downloads cpython-3.12.13-macos-aarch64-none. Confirm with `uv run --python 3.12 python --version` (expect `Python 3.12.x`). Python 3.12 MUST be installed before the next step (OpenHands requires exactly 3.12; system Python is 3.14 and will cause install failure — see RESEARCH Pitfall 3).
    2. `uv tool install openhands --python 3.12` — large package, may take several minutes.
    3. Confirm `openhands --version` prints `1.7.x` (or `openhands 1.7...`).

    PATH fallback: if `openhands` is not found after install, run `export PATH="$HOME/.local/bin:$PATH"` and append that line to `~/.zshrc` for persistence, then retry `openhands --version`.

    Do NOT run `openhands serve` in this task — that is Task 3 (a foreground/UI process needing human verification).
  </action>
  <verify>`uv run --python 3.12 python --version` prints `Python 3.12.x`; `openhands --version` prints `1.7.x`.</verify>
  <done>`openhands --version` returns 1.7.x and the binary resolves on PATH.</done>
</task>
