- Each `openhands` invocation is a long autonomous agent run on the LOCAL 35B model. 35B per-LLM-call ≈ 5.3s on this hardware (measured in v1+v1.1). Budget per task:
  - task1-scaffold: ~2–5 minutes (simple `cargo new` + listing)
  - task2-server: ~5–15 minutes (write src/main.rs; potentially iterates if agent hits borrow-checker thinking in advance — but task2 does NOT build, only writes)
  - task3-buildtest: ~5–25 minutes (this is where error-and-fix happens — build failures, possible runtime hangs, fixes, rebuilds)
- **Total run estimate: 15–40 minutes** (most likely range). If the run exceeds 40 minutes WITH NO PROGRESS visible in the JSONL, that is a RED FLAG — consider whether the agent is stuck in an irrecoverable loop and the scaffold fallback should be invoked.
- **Hard abort threshold: 60 minutes per task.** If any single openhands invocation has been running for >60 minutes (wall clock) and the JSONL shows no FinishAction and no new events for the past 5 minutes, kill the process, record the timeout in RUN-NOTES, and apply the scaffold fallback (for task2) or HALT-and-report (for task3).
- **The CLI exits 0 REGARDLESS of task success.** DO NOT judge outcome from exit code. Judge ONLY from the JSONL (FinishAction present, exit codes inside TerminalObservations, actual command output).
- **Run each invocation in the BACKGROUND** (run_in_background=true on the Bash tool) and POLL the JSONL until the run settles. Status pings every ~60s using the polling check below. Do NOT block the foreground on `sleep`.
- Outcomes are NOT pre-determined. The agent may write a working server, may hit BufReader-move E0382 and self-correct, may hit a read_to_string hang and self-correct (after task3's "if curl hangs" hint), may need scaffold fallback, or may produce an honest failure. Follow the decision rules below and record what ACTUALLY happens.

<jsonl_schema_reference>
Confirmed nested-dict schema from v1 + v1.1 runs (identical OpenHands version 1.16):

- **ActionEvent**: top-level `kind`=="ActionEvent", `id`, `timestamp`, `source` (MUST be "agent" for all agent actions). Nested `action.kind` is one of "TerminalAction" / "FinishAction"; `action.command` (the shell command for TerminalAction).
- **ObservationEvent**: top-level `kind`=="ObservationEvent", `action_id` (links back to the ActionEvent `id`), `timestamp`. Nested `observation.kind`=="TerminalObservation", `observation.exit_code`, `observation.content`, `observation.command`.
- **Error signals**: top-level `kind`=="ConversationErrorEvent"; or an AgentErrorEvent; or an observation `content` mentioning `file_editor` / `security_risk`.

**Reusable polling/status check (substitute the JSONL filename for TASK):**

```bash
python3 -c "
import json,collections
f='oh-workdir-rust/TASK.jsonl'
ev=[json.loads(l) for l in open(f) if l.strip().startswith('{')]
kinds=collections.Counter(e.get('kind') for e in ev)
finish=any(e.get('kind')=='ActionEvent' and e.get('action',{}).get('kind')=='FinishAction' for e in ev)
err=any(e.get('kind')=='ConversationErrorEvent' for e in ev)
ta=sum(1 for e in ev if e.get('kind')=='ActionEvent' and e.get('action',{}).get('kind')=='TerminalAction')
nonzero=sum(1 for e in ev if e.get('kind')=='ObservationEvent' and e.get('observation',{}).get('exit_code',0)!=0)
last_ts=ev[-1].get('timestamp','') if ev else ''
print('kinds=',dict(kinds),'| TerminalActions=',ta,'| FinishAction=',finish,'| ConvError=',err,'| NonZeroExits=',nonzero,'| last_ts=',last_ts)
"
```

**Poll loop pattern:** start an openhands invocation with `run_in_background=true`, then every ~60s run the status check above. The run is settled when EITHER FinishAction=True, OR ConvError=True, OR last_ts has not advanced for 3+ consecutive checks (~3 minutes idle). When settled, proceed to evaluate the result. Capture the background PID; if it overruns the 60-minute hard abort threshold, kill it.
</jsonl_schema_reference>
