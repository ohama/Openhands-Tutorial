take many minutes (Phase 2 measured ~14-15s per single tool-call cycle; these tasks are multi-cycle).
Model behavior is NOT deterministic — the exact error that surfaces, and how many retries are needed,
are empirical. Run each openhands invocation in the BACKGROUND (run_in_background) and poll its JSONL;
do NOT block a foreground call on a long agent loop. Apply the retry/adjust policy below. NEVER
fabricate an error-and-fix; if none emerges after retries, re-run for variance (CONTEXT).
</objective>

<invocation_template>
Every task uses this exact shape (PROMPT_FILE and TASK differ per task). Run in the background and
poll the JSONL; the CLI exits 0 in success AND failure, so completion is judged from the JSONL, not
the exit code:
```
OPENHANDS_SUPPRESS_BANNER=1 \
LLM_MODEL="openai/qwen-local" LLM_BASE_URL="http://127.0.0.1:4000/v1" LLM_API_KEY="dummy" \
OPENHANDS_WORK_DIR="/Users/ohama/projs/OpenHandsTests/oh-workdir" \
openhands --headless --json --yolo --override-with-envs \
  -t "$(cat /Users/ohama/projs/OpenHandsTests/.planning/phases/03-capture-the-openhands-run/task-prompts/<PROMPT_FILE>)" \
  2>/Users/ohama/projs/OpenHandsTests/oh-workdir/<TASK>.stderr.log \
  | tee /Users/ohama/projs/OpenHandsTests/oh-workdir/<TASK>.jsonl
```
After each invocation, judge the JSONL with the research B3/B4 parser (filter lines with
`l.strip().startswith('{')`):
- FinishAction present OR final agent MessageEvent => normal completion.
- `ConversationErrorEvent` `code=MaxIterationsReached` => iteration cap hit.
- No FinishAction + same command repeated 3+ times near the end => STUCK.
</invocation_template>

<retry_policy>
Per CONTEXT + research B7. For each task:
| Situation | Detected by (JSONL) | Action |
|-----------|---------------------|--------|
| Completed, intended files/result present | FinishAction + host-side check passes | Keep log; next task |
| MaxIterations | ConversationErrorEvent code=MaxIterationsReached | Re-run SAME prompt (fresh conversation sees the persisted files). Up to ~2x. |
| Stuck / looping | no FinishAction + repeated command | Re-run once; if still stuck, adjust THAT prompt to be more explicit and record the change in RUN-NOTES |
| Agent never built / wrong action | zero TerminalAction, or off-task | Rewrite prompt with a clearer action verb; record change |
| Still stuck after ~2 retries + 1 adjust | — | Record as blocker in RUN-NOTES and report; do NOT fake progress |
After each task, verify on the HOST (not just the JSONL) that the intended file now exists in
oh-workdir/calc (e.g. `test -f oh-workdir/calc/Lexer.fsl`). A fresh conversation can claim success
without the file actually being right; the host check is the source of truth.
</retry_policy>
