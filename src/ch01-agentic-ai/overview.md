# 에이전틱 AI 개요

현대 AI 시스템은 크게 두 가지 방식으로 동작합니다. 하나는 사용자의 질문에 한 번 답하고 멈추는 **반응형 챗봇(reactive chatbot)**이고, 다른 하나는 목표를 받아 스스로 계획을 세우고 완료될 때까지 행동을 반복하는 **에이전틱 AI(agentic AI)**입니다. 이 장에서는 두 방식의 차이를 명확히 이해하고, 왜 에이전틱 AI가 코드를 짜고, 빌드하고, 오류를 고치는 것처럼 복잡한 작업을 자동화할 수 있는지 살펴봅니다.

---

## 반응형 챗봇의 한계

반응형 챗봇은 다음과 같이 동작합니다.

1. 사용자가 질문이나 요청을 입력한다.
2. LLM(대형 언어 모델)이 그 입력을 읽고 텍스트 응답을 한 번 생성한다.
3. 대화가 끝난다.

챗봇은 텍스트를 만드는 데 탁월하지만, 그것이 전부입니다. 외부 파일을 직접 수정하거나, 터미널 명령을 실행하거나, 빌드 결과를 확인하는 일은 할 수 없습니다. 사람이 직접 LLM의 답을 읽고 각 단계를 일일이 실행해야 합니다. 즉, **챗봇은 텍스트 생성기**입니다 — 세상에 실제로 영향을 주지는 못합니다.

또한 챗봇은 이전 대화에서 어떤 일을 했는지 기억하는 "외부 상태"가 없습니다. 사용자가 맥락을 새로 제공하지 않으면, LLM은 이전 단계를 알 수 없습니다.

---

## 에이전틱 AI의 동작 방식

에이전틱 AI는 다릅니다. 사용자가 **목표(goal)**를 주면, 에이전트는 스스로 판단하며 목표가 달성될 때까지 **에이전트 루프(agent loop)**를 반복합니다.

루프의 각 반복(iteration)은 다음 순서로 이루어집니다.

1. **프롬프트 준비** — 지금까지의 이벤트 기록(메시지, 행동, 관찰)을 LLM에 전달할 형태로 정리한다.
2. **LLM 호출** — LLM이 다음에 무엇을 할지 결정한다(tool call 또는 메시지).
3. **행동(action) 실행** — 파일 편집, 터미널 명령 실행, 코드 빌드 등 실제 환경에 변화를 준다.
4. **관찰(observation) 수집** — 행동의 결과(표준 출력, 오류 메시지, 파일 내용 등)를 기록한다.
5. **반복 또는 종료** — 아직 목표에 도달하지 않았으면 1번으로 돌아간다.

이 구조에서 LLM은 "무엇을 다음에 할지 결정하는 두뇌" 역할을 하고, 실제 행동은 **툴 호출(tool calling)**을 통해 환경에 직접 영향을 줍니다. 사람이 매번 지시하지 않아도, 에이전트가 결과를 보고 스스로 다음 단계를 선택합니다.

---

## 핵심 차이 비교

| 항목 | 반응형 챗봇 | 에이전틱 AI |
|------|------------|------------|
| **응답 방식** | 한 번 응답 후 종료 | 목표 완료까지 루프 반복 |
| **환경에 대한 행동** | 텍스트만 생성 (행동 없음) | 파일 수정, 명령 실행 등 환경에 직접 행동 |
| **다음 단계 결정** | 사람이 매 단계 지시 | 에이전트가 스스로 다음 단계 결정 |
| **상태 관리** | 상태 없음 (각 응답 독립) | 진행 상황을 memory/context로 기억 |

이 네 가지 차이는 단순한 설계 선택이 아닙니다. 에이전트가 "코드를 짜라"는 목표 하나를 받아서, 탐색하고, 구현하고, 빌드하고, 오류를 수정해 완성품을 내놓는 것을 가능하게 하는 근본적인 구조입니다.

---

## 왜 에이전틱 AI가 중요한가

반응형 챗봇이 프로그래밍 질문에 코드 조각을 알려준다면, 에이전틱 AI는 실제로 코드를 파일에 쓰고, 빌드하고, 실패를 관찰하고, 스스로 고쳐 완성품을 만들어냅니다 — 사람의 개입 없이도요.

이 튜토리얼에서는 OpenHands가 로컬 Qwen LLM 서버를 두뇌 삼아, F# FsLex/FsYacc 계산기를 처음부터 끝까지 자율적으로 만들어내는 과정이 바로 이 원리의 실제 사례입니다.

이 차이를 실제로 경험하는 것이 이 튜토리얼의 핵심입니다. 에이전트가 어떻게 일하는지 개념을 먼저 이해해야, 4부에서 OpenHands가 왜 특정 행동을 선택했는지, 왜 어떤 단계에서 오류가 났고 어떻게 스스로 고쳤는지를 제대로 읽어낼 수 있습니다.

에이전틱 AI의 네 가지 핵심 용어 — **tool calling**, **agent loop**, **plan → write → test → run**, **memory/context window** — 는 다음 장 [핵심 개념과 용어](concepts.md)에서 자세히 정의합니다.
